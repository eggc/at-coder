スコアを G 以上にするために必要な最小の問題数を求めよ

データサイズ D はたかだか10個
pi はたかだか100個

問題を解いた数を K とする。
問題の選び方は p1, p2, ... p10 から K 個を選ぶやり方である。
K=1, ...1000 まで計算すれば確実に解答は求められる
しかし問題数が1000個あるときは、重複順列で計算して 10 ^ 1000 通りあるので計算不可能。

c が常に 0 であるような世界を考える。
このときは点数の高い順に問題を解けば、求める数はすぐ計算できる。

c が 0 ではない場合、点数の高い問題を解くよりも、
c を取りに行ったほうが良い点が出ることがある。

c を取りに行くかどうかを先に決め打ちしてから探索する。
c のとり方は 10種類の問題グループ p1, p2, ...p10 のどれを取るかで組み合わせが出せる。
選び方は 2^10 通り。これは全探索できる。

c のとり方が決まったらそれに準じた上で点数の高い順に問題を解けば最小値を計算できる。
こうすると各最小値が 2^10 通り得られるので、更にこれの最小値をとれば求める解となる。
