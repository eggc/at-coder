AB + CD = N を満たす a,b,c,d を求める。
N = 20000 なので A,B,C,D は 20000 以下。
3重ループで求められそうな気はするけど計算量的に無理。

1 を作れる AB + CD は存在しない
2 を作れる AB + CD は 1,1,1,1
3 を作れる AB + CD は 1,1,2,1 とか 1,2,1,1 とか無数にある

N = 20000 が与えられた時には x,y で 20000 x 20000 通り計算する
で、作ることのできる数リストを用意する。

1 -> 1 通り
2 -> 2 通り
3 -> 2 通り
4 -> 3 通り
5 -> 2 通り
6 -> 4 通り
7 -> 2 通り
8 -> 4 通り
...

ここでの計算量が 20000^2 = 4億なので無理だった。
ここの計算量減らす方法ないかな。
x * y <= N は常に成り立つから y <= N / x までなのか。
→この対応でかなり早くなったのでよさそう。

あとは

1 の場合の数 * 19999 の場合の数
2 の場合の数 * 19998 の場合の数
...
の総和を取る。これは 20000 * 2 ステップでできるだろう。
